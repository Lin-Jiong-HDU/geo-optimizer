## 概述

本文档描述 GEO 优化器的核心模块设计，包括内容评分系统、主优化器和优化策略实现。

## 1. pkg/analyzer/scorer.go - 内容评分系统

### 1.1 功能描述

对内容进行 GEO（生成式搜索引擎优化）质量评分，提供多维度分析和改进建议。

### 1.2 核心接口

```go
package analyzer

import (
    "context"
    "github.com/Lin-Jiong-HDU/geo-optimizer/pkg/llm"
    "github.com/Lin-Jiong-HDU/geo-optimizer/pkg/models"
)

// Scorer 内容评分器
type Scorer struct {
    llmClient llm.LLMClient
}

// NewScorer 创建评分器
func NewScorer(client llm.LLMClient) *Scorer

// Score 对内容进行GEO评分
func (s *Scorer) Score(ctx context.Context, content string) (*models.GeoScore, error)

// ScoreWithAnalysis 对内容进行评分并返回详细分析
func (s *Scorer) ScoreWithAnalysis(ctx context.Context, content string) (*models.ContentAnalysis, error)

// Compare 对比优化前后的评分
func (s *Scorer) Compare(ctx context.Context, before, after string) (*ScoreComparison, error)
```

### 1.3 评分维度

| 维度 | 权重 | 评估标准 |
|------|------|----------|
| Structure | 20% | 标题层级、段落结构、列表使用、章节划分 |
| Authority | 25% | 数据引用、来源标注、作者信息、专业术语 |
| Clarity | 20% | 语言简洁度、逻辑连贯性、可读性 |
| Citation | 25% | 可引用的结论、事实陈述、数据支持 |
| Schema | 10% | 结构化数据完整性、JSON-LD 标记 |

### 1.4 实现逻辑

#### 方案一：基于 LLM 的智能评分（推荐）

```go
func (s *Scorer) ScoreWithAnalysis(ctx context.Context, content string) (*models.ContentAnalysis, error) {
    // 直接调用 LLM 客户端的分析功能
    analysis, err := s.llmClient.AnalyzeContent(ctx, content)
    if err != nil {
        return nil, err
    }
    return analysis, nil
}
```

#### 方案二：混合评分（LLM + 规则）

```go
func (s *Scorer) Score(ctx context.Context, content string) (*models.GeoScore, error) {
    // 1. 基础规则评分（快速）
    ruleScore := s.scoreByRules(content)

    // 2. LLM 深度分析（准确）
    llmScore, err := s.llmClient.AnalyzeContent(ctx, content)
    if err != nil {
        // 降级到规则评分
        return ruleScore, nil
    }

    // 3. 融合结果（加权平均）
    return s.mergeScores(ruleScore, llmScore), nil
}

func (s *Scorer) scoreByRules(content string) *models.GeoScore {
    score := &models.GeoScore{}

    // 结构评分：检查标题、段落、列表
    score.Structure = s.calculateStructureScore(content)

    // 权威性：检查引用、数据、来源
    score.Authority = s.calculateAuthorityScore(content)

    // 清晰度：检查句子长度、段落长度
    score.Clarity = s.calculateClarityScore(content)

    // 可引用性：检查结论、事实陈述
    score.Citation = s.calculateCitationScore(content)

    // Schema：检查是否包含 JSON-LD
    score.Schema = s.calculateSchemaScore(content)

    return score
}
```

## 2. pkg/optimizer/optimizer.go - 主优化器

### 2.1 功能描述

作为优化流程的编排器，协调多个优化策略，调用 LLM 生成优化内容。

### 2.2 核心接口

```go
package optimizer

import (
    "context"
    "github.com/Lin-Jiong-HDU/geo-optimizer/pkg/llm"
    "github.com/Lin-Jiong-HDU/geo-optimizer/pkg/models"
    "github.com/Lin-Jiong-HDU/geo-optimizer/pkg/analyzer"
)

// Optimizer GEO优化器
type Optimizer struct {
    llmClient     llm.LLMClient
    scorer        *analyzer.Scorer
    strategies    map[models.StrategyType]Strategy
}

// New 创建优化器
func New(client llm.LLMClient) *Optimizer

// Optimize 执行优化
func (o *Optimizer) Optimize(ctx context.Context, req *models.OptimizationRequest) (*models.OptimizationResponse, error)

// OptimizeWithStrategy 执行特定策略优化
func (o *Optimizer) OptimizeWithStrategy(ctx context.Context, req *models.OptimizationRequest, strategy models.StrategyType) (*models.OptimizationResponse, error)
```

### 2.3 优化流程

```
┌─────────────────────────────────────────────────────────────┐
│                    Optimizer.Optimize()                      │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
        ┌───────────────────────────────────────┐
        │  1. 参数验证与默认值设置                │
        │     - 检查必填字段                      │
        │     - 设置默认策略                       │
        └───────────────────────────────────────┘
                            │
                            ▼
        ┌───────────────────────────────────────┐
        │  2. 内容评分（优化前）                  │
        │     scorer.Score(originalContent)      │
        └───────────────────────────────────────┘
                            │
                            ▼
        ┌───────────────────────────────────────┐
        │  3. 策略选择与执行                     │
        │     for each strategy in req.Strategies │
        │       - strategy.Apply(content, req)   │
        │       - content = result.Content       │
        └───────────────────────────────────────┘
                            │
                            ▼
        ┌───────────────────────────────────────┐
        │  4. 调用 LLM 生成优化内容              │
        │     llmClient.GenerateOptimization()   │
        └───────────────────────────────────────┘
                            │
                            ▼
        ┌───────────────────────────────────────┐
        │  5. 内容评分（优化后）                  │
        │     scorer.Score(optimizedContent)     │
        └───────────────────────────────────────┘
                            │
                            ▼
        ┌───────────────────────────────────────┐
        │  6. 解析与构建响应                      │
        │     - 提取 Schema                       │
        │     - 提取 FAQ                          │
        │     - 分析产品植入                      │
        │     - 计算评分提升                      │
        └───────────────────────────────────────┘
                            │
                            ▼
                    返回 OptimizationResponse
```

### 2.4 实现逻辑

```go
func (o *Optimizer) Optimize(ctx context.Context, req *models.OptimizationRequest) (*models.OptimizationResponse, error) {
    // 1. 参数验证
    if err := o.validateRequest(req); err != nil {
        return nil, err
    }

    // 2. 设置默认值
    o.setDefaults(req)

    // 3. 优化前评分
    scoreBefore, _ := o.scorer.Score(ctx, req.Content)

    // 4. 应用策略预处理（可选）
    content := req.Content
    for _, strategyType := range req.Strategies {
        if strategy, ok := o.strategies[strategyType]; ok {
            content = strategy.Preprocess(content, req)
        }
    }

    // 更新请求内容
    req.Content = content

    // 5. 调用 LLM 生成优化内容
    optResp, err := o.llmClient.GenerateOptimization(ctx, req)
    if err != nil {
        return nil, err
    }

    // 6. 优化后评分
    scoreAfter, _ := o.scorer.Score(ctx, optResp.OptimizedContent)

    // 7. 使用 parser 解析 LLM 返回内容
    parser := llm.NewParser()

    response := &models.OptimizationResponse{
        OptimizedContent:      optResp.OptimizedContent,
        Title:                optResp.Title,
        SchemaMarkup:         parser.extractSchema(optResp.OptimizedContent),
        FAQSection:           parser.extractFAQ(optResp.OptimizedContent),
        Summary:              parser.extractSummary(optResp.OptimizedContent),
        ProductMentions:      parser.extractProductMentions(optResp.OptimizedContent, req.Enterprise.ProductName),
        MentionCount:         parser.countProductMentions(optResp.OptimizedContent, req.Enterprise.ProductName),
        DifferentiationPoints: parser.extractDifferentiationPoints(optResp.OptimizedContent),
        AppliedStrategies:    req.Strategies,
        ScoreBefore:          scoreBefore.OverallScore(),
        ScoreAfter:           scoreAfter.OverallScore(),
        GeneratedAt:          optResp.GeneratedAt,
        LLMModel:             optResp.LLMModel,
        TokensUsed:           optResp.TokensUsed,
        Version:              "1.0.0",
    }

    return response, nil
}
```

```go
func (s *SchemaStrategy) BuildPrompt(req *models.OptimizationRequest) string {
    schemaType := s.inferSchemaType(req)

    return fmt.Sprintf(`请为以下内容生成 JSON-LD 格式的 Schema.org 标记：

【内容信息】
标题：%s
内容：%s
企业：%s
产品：%s

【Schema类型】%s

请只返回 JSON 格式的 Schema 标记，使用 <script type="application/ld+json"> 包裹。`,
        req.Title, req.Content,
        req.Enterprise.CompanyName,
        req.Enterprise.ProductName,
        schemaType)
}

func (s *SchemaStrategy) inferSchemaType(req *models.OptimizationRequest) string {
    // 根据内容推断 Schema 类型
    if strings.Contains(req.Content, "如何") || strings.Contains(req.Content, "教程") {
        return "HowTo"
    }
    if strings.Contains(req.Content, "产品") || strings.Contains(req.Content, "购买") {
        return "Product"
    }
    if strings.Contains(req.Content, "文章") || strings.Contains(req.Content, "指南") {
        return "Article"
    }
    return "WebPage"
}
```

## 5. 错误处理

```go
import "errors"

var (
    ErrInvalidRequest    = errors.New("invalid optimization request")
    ErrNoStrategy       = errors.New("no strategy specified")
    ErrLLMFailed        = errors.New("LLM generation failed")
    ErrParseFailed      = errors.New("failed to parse LLM response")
)

func (o *Optimizer) validateRequest(req *models.OptimizationRequest) error {
    if req.Content == "" {
        return fmt.Errorf("%w: content is required", ErrInvalidRequest)
    }
    if req.Enterprise.CompanyName == "" {
        return fmt.Errorf("%w: enterprise info is required", ErrInvalidRequest)
    }
    if len(req.Strategies) == 0 {
        // 使用默认策略
        req.Strategies = []models.StrategyType{
            models.StrategyStructure,
            models.StrategySchema,
        }
    }
    return nil
}
```

### 9.2 缓存机制

```go
type CachedScorer struct {
    scorer  *analyzer.Scorer
    cache   *lru.Cache
}

func (c *CachedScorer) Score(ctx context.Context, content string) (*models.GeoScore, error) {
    // 计算内容哈希
    key := sha256.Sum256([]byte(content))

    // 检查缓存
    if cached, ok := c.cache.Get(key); ok {
        return cached.(*models.GeoScore), nil
    }

    // 调用底层评分器
    score, err := c.scorer.Score(ctx, content)
    if err != nil {
        return nil, err
    }

    // 存入缓存
    c.cache.Add(key, score)
    return score, nil
}
```

| 模块 | 文件 | 核心功能 |
|------|------|----------|
| Scorer | scorer.go | 内容质量评分，多维度分析 |

通过清晰的接口设计和模块化架构，系统易于扩展和维护。
